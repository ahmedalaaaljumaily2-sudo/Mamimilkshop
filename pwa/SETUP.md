# خطوات تشغيل تطبيق مخزون حليب الأطفال

التطبيق يحتاج خطوتين فقط قبل ما يشتغل: (1) ربطه بقاعدة بيانات مجانية حتى يشترك فيها الشخصين، و(2) نشره على رابط دائم حتى تقدروا تفتحوه من الموبايلين.

---

## الخطوة 1: إنشاء قاعدة بيانات مجانية (Firebase) — 5 دقائق

1. روح إلى https://console.firebase.google.com وسجّل دخول بحساب Google.
2. اضغط **"Add project" / إضافة مشروع**، وسمّه أي اسم مثلاً `milk-store`. تقدر تلغي Google Analytics (غير ضروري).
3. من القائمة الجانبية اليسرى، اختر **Build → Firestore Database**.
4. اضغط **Create database**. اختر أي موقع سيرفر قريب منك، واختر **Start in production mode**.
5. بعد الإنشاء، افتح تبويب **Rules** بالأعلى واستبدل المحتوى بهذا:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /store/{docId} {
      allow read, write: if true;
    }
    match /sales/{saleId} {
      allow read, write: if true;
    }
  }
}
```

ثم اضغط **Publish**. (هذا يسمح بالقراءة/الكتابة فقط على مستندات المخزون والشركات والمبيعات الخاصة بالتطبيق، وليس على كل قاعدة البيانات.)

6. ارجع لصفحة المشروع الرئيسية، واضغط على أيقونة **</> (Web)** لإضافة تطبيق ويب جديد. اعطه أي اسم واضغط **Register app**.
7. راح يظهر لك كود فيه كائن اسمه `firebaseConfig` يشبه:

```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "milk-store-xxxx.firebaseapp.com",
  projectId: "milk-store-xxxx",
  storageBucket: "milk-store-xxxx.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

انسخ هذا الكائن بالكامل.

8. افتح ملف **index.html** (بمحرر نصوص عادي)، ولاقي هذا الجزء قريب من الأعلى:

```js
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  ...
};
```

واستبدله بالكامل بالكود اللي نسخته من Firebase. احفظ الملف.

---

## الخطوة 2: نشر التطبيق على رابط دائم

أسهل طريقة بدون أي خبرة برمجية:

1. روح إلى https://app.netlify.com/drop
2. اسحب مجلد التطبيق كامل (اللي فيه index.html و manifest.json و sw.js ومجلد icons) وأفلته في الصفحة.
3. خلال ثوانٍ راح يعطيك رابط جاهز مثل `https://your-app-name.netlify.app`.
4. هذا هو الرابط اللي تفتحونه إنت والشخص الثاني.

> بديل: لو تعرف تستخدم GitHub، تقدر ترفع نفس الملفات إلى مستودع (repository) وتفعّل **GitHub Pages** من إعدادات المستودع (Settings → Pages)، وراح يعطيك رابط مشابه.

---

## الخطوة 3: تثبيت التطبيق على الموبايلين

1. افتح الرابط من متصفح الموبايل (كروم بالأندرويد، سفاري بالآيفون).
2. من قائمة المتصفح (⋮ أو زر المشاركة ⬆️)، اختر **"إضافة إلى الشاشة الرئيسية / Add to Home Screen"**.
3. راح تظهر أيقونة التطبيق (البروتيرة) على الشاشة الرئيسية، وتفتح بملء الشاشة بدون شريط المتصفح.
4. كرر هذا على جهاز الشخص الثاني بنفس الرابط بالضبط — وبكذا يشتركون بنفس المخزون لحظياً.

---

## أقسام التطبيق

- **🔍 مسح المادة**: مسح/بحث عن منتج، إضافة منتج جديد (مع تحديد الشركة وسعر الشراء وسعر البيع)، تسجيل شراء جديد (Restock)، وتعديل الشركة/الأسعار لاحقاً.
- **💰 المبيعات**: مسح باركود لتسجيل عملية بيع (بسعر بيع قابل للتعديل لحظة البيع)، مع قائمة بكل عمليات بيع اليوم وإجمالي المبيعات، وإمكانية استعراض أي يوم سابق.
- **📦 المخزون**: يعرض المنتجات مجمّعة كمجموعات قابلة للطي حسب شركة الحليب، وتظهر فيها الكمية وتاريخ الانتهاء وسعري الشراء والبيع لكل منتج.

جميع التواريخ بالتطبيق تُعرض بصيغة أرقام فقط (يوم/شهر/سنة).

## ملاحظات مهمة

- إذا نسيت تعبّي `firebaseConfig`، التطبيق راح يعرض رسالة توضح إنك بحاجة تكمل الإعداد.
- التطبيق يحتاج اتصال إنترنت لقراءة/حفظ المخزون وتشغيل الكاميرا.
- ما فيه أي تكلفة على هذا الاستخدام — كل من Firebase (بحدود سخية جداً للاستخدام الشخصي) و Netlify مجانيين بالكامل لحجم هذا التطبيق.
- إذا احتجت لاحقاً تغيّر التصميم أو تضيف ميزة، عدّل ملف index.html وأعد رفعه بنفس الطريقة (بالسحب والإفلات على Netlify، أو ادفع تحديثاً جديداً على GitHub).
